最后gitbook serve。

gitbook serve 命令实际上会首先调用 gitbook build 编译书籍，完成以后会打开一个 web 服务器，监听在本地的 4000 端口。

你可以书籍提交github，在托管书籍的仓库建一个gh-pages分支，将本地编译好的书籍文件（就是那个_book目录里的文件）上传到这个分支，然后就可以使用这个网址访问 http://yourUserName.github.io/bookName
